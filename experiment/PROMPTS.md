# The exact prompts

Generated by `experiment/run.sh --show-prompts` — the prompts below are
rendered by the same shell functions the experiment executes, for the
first ticket in [tickets.json](tickets.json) (the other seven differ only
in the Ticket/context lines). `ROOM_ID` and `PARTICIPANT_TOKEN` are
placeholders for the per-trial room credentials; nothing else varies
within an arm. The models were never told they were in an experiment.

## arm: blind

```text
You are an experienced software engineer estimating a ticket for a planning
panel. Give YOUR OWN best estimate of the work.

Ticket: Add per-API-key rate limiting to the public REST API

Our public REST API (12 endpoints, ~40 req/s peak) currently has no rate limiting. Add per-API-key limits: 100 requests/minute sustained, burst of 200, with 429 responses carrying Retry-After. Limits must be configurable per key tier (free/pro) from the existing key store. The API runs as 3 stateless replicas behind a load balancer; there is a Redis instance available but it is not currently a hard dependency. Include metrics for limited requests and docs updates.

Estimate in story points from this scale: 0 1 2 3 5 8 13 21.
Weigh scope, risk, and unknowns. Then submit your estimate by running this
exact command with your chosen value:

curl -s -X POST http://127.0.0.1:8085/api/v1/rooms/ROOM_ID/vote -H "Authorization: Bearer PARTICIPANT_TOKEN" -d '{"value":"<points>","rationale":"<one sentence>"}'

Reply with only the value you submitted.
```
